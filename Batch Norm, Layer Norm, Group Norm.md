
# 归一化技术 Batch Norm, Layer Norm, Group Norm

## 1. BN原理：为什么有效？

- **核心思想**：解决**内部协变量偏移（Internal Covariate Shift）**——训练中网络层输入分布随时间变化导致训练慢、梯度不稳定。
    
- **操作步骤（对每一层输入）**：
    
    1. 计算当前batch的均值 $\mu$ 和方差 $\sigma^2$；
        
    2. **归一化**：；
        
    3. **缩放平移**：( （$\gamma/\beta$ 为可学习参数，保留表征能力）。
        
- **收益**：
    
    - 允许使用更大学习率（训练提速）；
        
    - 减弱梯度消失/爆炸（梯度传递更稳定）；
        
    - 隐式正则化效果（减少对 Dropout 的依赖）。
        

## 2. 训练期 vs 推理期统计量的关键区别

|**阶段**|**统计量来源**|**为什么这样设计？**|
|---|---|---|
|**训练**|使用当前batch的 $\mu$ 和 $\sigma^2$|实时适应数据分布，同时更新全局滑动平均统计量（$\mu_{global}$, $\sigma^2_{global}$）|
|**推理**|使用训练集全局滑动平均的 $\mu_{global}$ 和 $\sigma^2_{global}$|避免依赖单样本或 batch 的随机性，保证输出稳定性和一致性|

> **注**：滑动平均通过指数移动平均（EMA）更新，例如：
> 
> $(\mu_{global} = momentum \cdot \mu_{global} + (1 - momentum) \cdot \mu_{batch})$
> 
> （通常 $momentum=0.9 \sim 0.99$）

## 3. BN vs LN vs GN：取舍与使用场景

|**方法**|**归一化维度**|**适用场景**|**优缺点**|
|---|---|---|---|
|**BN**|Batch × Height × Width（每通道单独归一化）|- 大batch（如 >32）的CNN任务（图像分类）<br><br>  <br><br>- 稳定且均匀的数据分布|✅ 加速收敛效果好<br><br>  <br><br>❌ batch小时性能崩坍（方差估算不准）|
|**LN**|样本内所有特征（Layer内）|- 变长序列（NLP/RNN/Transformer）<br><br>  <br><br>- 小batch或动态图结构|✅ 不依赖batch大小<br><br>  <br><br>❌ 对CNN空间结构不敏感|
|**GN**|将通道分组，组内归一化|- 小batch任务（目标检测/分割）<br><br>  <br><br>- 替代BN when batch < 8|✅ 在batch=2时仍有效<br><br>  <br><br>❌ 分组需调参|

### 实战选择原则：

- **图像分类（ResNet）**：优先 BN（batch ≥ 32）；
    
- **自然语言处理（BERT/Transformer）**：优先 LN（对序列长度不变性）；
    
- **低内存场景（目标检测YOLO）**：batch 小，用 GN 替代 BN。
    

## 4. 联合使用收益案例

- **CNN目标检测任务（batch=4）**：
    
    - 使用 BN：训练不稳定，mAP 波动大（±3%）；
        
    - 切换为 GN（group=8）：mAP 提升 2.5%，训练曲线平滑。
        
- **Transformer模型**：
    
    - 使用 BN：序列长度变化导致性能下降；
        
    - 使用 LN：训练加速且泛化稳定（困惑度降低 0.5）。
